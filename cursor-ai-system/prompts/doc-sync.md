# Documentation Sync Template

## Purpose
Template for keeping documentation synchronized with code changes.

## When to Use
- After code changes that affect functionality
- When architecture changes
- When API contracts change
- When adding new features

## Documentation Files to Update

### README.md
- Project description
- Setup instructions
- Usage examples
- Configuration options

### CHANGELOG.md
- New features
- Bug fixes
- Breaking changes
- Deprecations

### Architecture Documentation
- System architecture
- Component diagrams
- Data flow
- Design decisions

### API Documentation
- Endpoint descriptions
- Request/response formats
- Authentication
- Error codes

### Developer Documentation
- Setup guide
- Coding conventions
- Testing guide
- Contribution guidelines

## Sync Process

### Step 1: Identify Changes
1. Review code changes
2. Identify affected documentation
3. Check for new features
4. Note breaking changes

### Step 2: Update Documentation
1. Update relevant sections
2. Add new sections if needed
3. Remove outdated information
4. Fix broken links

### Step 3: Verify Accuracy
1. Check examples work
2. Verify links
3. Review formatting
4. Test instructions

### Step 4: Commit
1. Stage documentation changes
2. Commit with code changes
3. Reference related code changes
4. Update version if needed

## Documentation Standards

### Clarity
- Use clear, concise language
- Avoid jargon
- Include examples
- Explain "why" not just "what"

### Completeness
- Cover all features
- Include edge cases
- Document errors
- Provide troubleshooting

### Accuracy
- Keep up-to-date
- Test examples
- Verify instructions
- Review regularly

### Consistency
- Follow style guide
- Use consistent terminology
- Maintain structure
- Keep formatting consistent

## Example Documentation Update

```markdown
## Documentation Updates

### README.md
- Added new authentication section
- Updated setup instructions
- Added environment variables

### CHANGELOG.md
- Added v1.2.0 entry
- Documented new auth feature
- Noted breaking changes

### docs/ARCHITECTURE.md
- Updated auth flow diagram
- Added new component descriptions
- Updated data model
```

## Maintenance

- Review documentation monthly
- Update with each release
- Fix issues as found
- Keep examples current
