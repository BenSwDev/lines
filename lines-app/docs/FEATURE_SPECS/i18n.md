# Feature Specification: Internationalization (i18n)

**Status:** Implemented  
**Library:** next-intl  
**Version:** v1.1.0

---

## Overview

Full internationalization support with:

- Locale-based routing
- Translation files for all UI strings
- RTL support for Hebrew
- Ready for additional languages

---

## Supported Locales

| Locale | Language | Direction | Status      |
| ------ | -------- | --------- | ----------- |
| `he`   | עברית    | RTL       | ✅ Complete |
| `en`   | English  | LTR       | ✅ Complete |

Default locale: **Hebrew** (`he`)

---

## Routing Structure

All routes are now locale-prefixed:

```
/he/                          → Hebrew home
/en/                          → English home
/he/venues/[id]/info          → Hebrew venue info
/en/venues/[id]/info          → English venue info
/he/auth/login                → Hebrew login
/en/auth/login                → English login
```

---

## Translation Files

Located in `messages/`:

- `messages/en.json` - English translations
- `messages/he.json` - Hebrew translations

### Structure

```json
{
  "common": {
    /* buttons, actions */
  },
  "auth": {
    /* login, register */
  },
  "venues": {
    /* venue management */
  },
  "venueInfo": {
    /* contact details */
  },
  "workspace": {
    /* sidebar, tabs */
  },
  "lines": {
    /* lines management */
  },
  "calendar": {
    /* calendar UI */
  },
  "settings": {
    /* venue settings */
  },
  "errors": {
    /* error messages */
  }
}
```

---

## Usage in Components

```typescript
import { useTranslations } from "next-intl";

export function MyComponent() {
  const t = useTranslations("venues");
  return <h1>{t("title")}</h1>;
}
```

---

## Adding New Languages

1. Create `messages/{locale}.json`
2. Add locale to `src/i18n/config.ts`:
   ```ts
   export const locales = ["en", "he", "ar"] as const;
   ```
3. Add locale name to `localeNames`
4. All routes automatically available
